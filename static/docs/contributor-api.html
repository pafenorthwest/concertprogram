APIs supporting Contributor updates supporting multiple roles including 'composers'

create a new contributor

Adds a new contributor with role and years active. External clients must present an Authorization bearer token; requests from the Concert Program app may also provide the pafe_auth cookie issued by the service. If the role is omitted or not recognized, it is set to Composer. The optional alias value is stored in the notes field.

Authorizations:
BearerAuth
Request Body schema: application/json
required

JSON data for the contributor to create

full_name
required
string
years_active
required

role
string
Default: "Composer"
Enum: "Composer" "Copyist" "Editor" "Arranger" "Transcriber" "Realizer" "Orchestrator"
